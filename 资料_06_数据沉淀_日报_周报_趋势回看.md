## 来源

- 作者：苹果落地
- 来源：人间折腾录
- 日期：2026 年 4 月 17 日

---

## 一、本篇目标

解决**复盘问题**：每次结束有日报，每周有周报，趋势可回看。

**三个验收点**：

| 维度 | 回答的问题 |
|------|-----------|
| 日报 | 今天完成了什么？卡在哪里？明天最该抓什么？ |
| 周报 | 本周完成度有无变化？哪些问题重复出现？下周只抓哪个重点？ |
| 趋势 | 最近几周关键指标走势（不靠聊天记录判断） |

> 重点不是做"好看的总结"，而是把前几篇积累的数据整理成持续可用的复盘资产。

---

## 二、为什么必须做

如果没有日报和周报，系统会出现三个明显问题：

1. 每次学习结束后，没有结果沉淀 → 家长凭印象判断
2. 后面的组合计划没有可参考的数据输入
3. 系统只是一次次即时对话，不是可复盘的系统

> 把系统从"会执行"推进到"能积累"。

---

## 三、技术栈与依赖

| 组件 | 职责 |
|------|------|
| `skills/daily-report` | 生成日报并落盘 |
| `skills/weekly-report` | 生成周报 + 趋势查询 |
| `data/reports/daily/` | 日报归档 |
| `data/reports/weekly/` | 周报归档 |
| `data/reports/trends/trends.csv` | 趋势数据源 |

**依赖数据（最小集）**：
- `data/routines/state.json`：会话、时长、轮次、确认状态
- `data/homework/parsed/`：当天任务计划
- `data/mistakes/` 或 `data/qas/`：新增错题/疑问

> 先保证日报能稳定生成，周报和趋势再在此基础上汇总。

---

## 四、目录结构

```
data/reports/
├── daily/        # 日报
├── weekly/       # 周报
└── trends/
    └── trends.csv  # 趋势数据
```

> 不要把日报、周报混在聊天记录或临时文件里，复盘数据要有明确归档位置。

---

## 五、日报模板

### 命名规范
```
data/reports/daily/YYYY-MM-DD_member_id_daily.md
```
**member_id 必须带上**，多孩子场景不能省。

### 日报模板结构

```markdown
# 日报 YYYY-MM-DD (member_id)

## 今日完成
- （列 1-5 条最关键完成项）

## 今日卡点（只写 1 个）
- 卡点：
- 可能原因（可选）：

## 明日 1 个重点（只写 1 个）
- 明日重点：

## 数据摘要（自动生成）
- 学习时长（分钟）：
- 番茄钟轮次：
- 任务完成度（完成/总数）：
- 新增错题/疑问数：
```

**最重要的是三段**：今日完成、今日卡点、明日 1 个重点
> 家长真正用到的通常不是长串描述，而是这三个结论。

---

## 六、周报模板

**周报职责不是重复日报，而是做聚合和变化判断。**

### 命名规范
```
data/reports/weekly/YYYY-Www_member_id_weekly.md
```

### 周报模板结构

```markdown
# 周报 YYYY-Www (member_id)

## 本周概览
- 总学习时长（分钟）：
- 番茄钟总轮次：
- 完成任务数（完成/总数）：

## 完成度趋势（对比上周）
- 时长：↑/↓（变化值）
- 轮次：↑/↓
- 完成度：↑/↓

## 薄弱点趋势（最简三条）
1) Top 卡点/错因：
2) Top 卡点/错因：
3) Top 卡点/错因：

## 下周只抓 1 个重点
- 重点：
- 建议策略：
```

> 不要写成流水账，重点就是：本周整体情况 + 上周对比 + 最重复的问题 + 下周只抓一个重点。

---

## 七、趋势字段（最小可用）

`data/reports/trends/trends.csv`：

```csv
date,member_id,minutes,rounds,homework_done,homework_total,questions_new,mistakes_new,top_blocker,plan_name
```

| 字段 | 含义 |
|------|------|
| `date` | 日期 |
| `member_id` | 成员 ID |
| `minutes` | 当天学习总时长 |
| `rounds` | 当天番茄钟轮次 |
| `homework_done` / `homework_total` | 任务完成度 |
| `questions_new` | 新增疑问数 |
| `mistakes_new` | 新增错题数 |
| `top_blocker` | 当天最主要卡点 |
| `plan_name` | 当天使用的计划模板 |

> 思路不是"先设计一个完美趋势系统"，而是**先把每天最关键的一行数据写下来**。有了这份 CSV，后面做/趋势 4 周、图表、看板都有统一数据源。

## 八、daily-report Skill

**路径**：`skills/daily-report/SKILL.md`

```markdown
## 指令
- /日报
- /日报 日期=YYYY-MM-DD（可选）

## 身份
- 用 openid → member_id → role（来自 index.json）
- 默认生成"发送者本人"的日报
- admin 可指定 member_id=<...> 查看摘要

## 数据源（最小）
- data/routines/state.json
- data/homework/parsed/
- data/mistakes/ 或 data/qas/

## 输出
- 群内：3 行摘要 + "已生成日报文件"
- 文件：data/reports/daily/YYYY-MM-DD_member_id_daily.md

## 趋势（可选）
- 追加 data/reports/trends/trends.csv 一行
```

> 关键不是"写多漂亮"，而是**格式和数据源稳定**。

---

## 九、weekly-report Skill

**路径**：`skills/weekly-report/SKILL.md`

```markdown
## 指令
- /周报
- /周报 周=YYYY-Www（可选）
- /趋势 4 周

## 身份与权限
- kid：只能看自己摘要
- parent：可看自己 + linked_children 的摘要
- admin：可看家庭汇总摘要

## 数据源
- data/reports/daily/
- data/reports/trends/trends.csv
- data/mistakes/subjects 或 qas（可选增强）

## 输出
- 周报文件：data/reports/weekly/YYYY-Www_member_id_weekly.md
- 群内：周报摘要 + 文件已生成提示
- /趋势 4 周：输出每周关键指标
```

**权限收口要注意**：
- 孩子只看自己的摘要
- 家长看自己和已绑定孩子
- 管理员看家庭级摘要

> 不要因为能聚合数据，就把隐私边界做散了。

## 十、落地策略：先手动，再自动

**最稳落地方式**：

| 方案 | 步骤 |
|------|------|
| 方案 A：先手动 | 学习结束后，群里手动发 `/日报`，验证数据源和落盘 |
| 方案 B：再自动 | 稳定后，把 daily-report 接到结束动作上：`/结束学习` 后自动触发 |

> 不要一开始就追求全自动。先把日报稳定生成，比什么都重要。

---

## 十一、验收步骤

### Step 1：完成一次完整学习
```
/开始学习...
/结束学习
```
确保本次学习有时长、轮次和任务数据。

### Step 2：生成日报
```
群发：/日报
```
**检查**：
- `data/reports/daily/` 下出现当天日报文件
- 群内返回日报摘要

### Step 3：生成周报
```
群发：/周报
```
**检查**：
- `data/reports/weekly/` 下出现本周周报文件
- 群内返回周报摘要

### Step 4：查看趋势
```
群发：/趋势 4 周
```
即使数据不满 4 周，也应至少能输出已有周的数据汇总。

---

## 十二、常见问题

| 问题 | 排查 |
|------|------|
| 日报生成了但内容是空的 | 检查：state.json 是否写了开始/结束/轮次/时长 / homework/parsed/ 是否存在 / 错题目录是否有数据 |
| 趋势输出不了 | trends.csv 还没创建 / 字段不一致导致聚合失败 / 每次生成日报时顺手 append 一行趋势数据 |
| 多孩子报告串台 | 文件名必须带 member_id / 默认 target=发送者本人 / 家长代查要显式指定对象 |

---

## 十三、小结

**本篇完成后，学习系统就从"会执行"进入了"可复盘"。**

**沉淀出三层结果**：

| 层级 | 内容 |
|------|------|
| 当天日报 | 完成 / 卡点 / 明日重点 |
| 每周周报 | 概览 + 趋势对比 + 薄弱点 + 下周重点 |
| 趋势数据 | 可回看的关键指标 CSV |

**这一步真正补上的是一个"系统的长期记忆"。**

> 后面的组合计划、强度调节、升级阈值优化，最终都要依赖这些数据。

- 作者：苹果落地
- 来源：人间折腾录
- 日期：2026 年 4 月 14 日

## 核心问题定义

### 传统 AI 学习系统的痛点
- 停留在：拆作业、讲题、给建议、做总结
- 前提假设：用户会主动执行

### 真实情况
- 不开始
- 没节奏
- 中途打断
- 没收尾

### 本质问题
> **不是"AI 不够聪明"，而是：缺少一条从"指令"到"执行"的完整链路。**

---

## 三层架构

### 1️⃣ 学习主干层（内容系统）

**职责**：把输入变成结构化任务，沉淀学习数据

**核心链路**：
```
作业/阅读 → 拆解 → 双视图 → 落盘
```

**关键设计**：同一任务，按角色输出不同信息密度
- **孩子版**：短句 + 可执行 + 可打勾（5-20 分钟粒度）
- **家长版**：顺序 + 估时 + ≤3 检查点

**数据沉淀**：
| 目录 | 内容 |
|------|------|
| homework/inbox | 原始输入 |
| homework/parsed | 拆解结果 |
| reports/daily | 日报数据 |
| reports/weekly | 周报数据 |

---

### 2️⃣ 智能家居执行层（执行系统） ⭐核心升级

**职责**：把"建议"变"动作"

**设备分工**：
| 设备 | 功能 |
|------|------|
| 音响 | 播报（开始/节奏/结束） |
| 台灯 | 状态指示（学习/休息/夜间） |
| Timer | 节奏推进（番茄钟） |

**执行链路**：
```
群里发"开始学习"
  → OpenClaw 识别意图
  → 触发 study-mode Skill
  → 调用 Home Assistant
  → 音响播报 + 台灯切换 + Timer 启动
  → 自动推进节奏
```

**关键变化**：学习从"看消息"变成"环境驱动"

---

### 3️⃣ 家长组合计划层（策略系统）

**职责**：解决"灵活性"和"可控性"的平衡

**结构**：
```
模板 + 模块 + 参数
```

**模板类型**：轻量 / 标准 / 冲刺 / 共读

**模块类型**：启动 / 节奏 / 升级 / 收尾

**参数**：配置驱动行为，不写死 Skill

---

## 三条核心闭环

### 闭环①：开始 / 节奏 / 结束
```
触发：开始学习
  → 执行：音响 + 灯 + timer
  → 推进：自动节奏
  → 结束：确认完成验收
```

### 闭环②：未确认升级
```
未确认 → 轻提醒
        → 仍未确认 → 家长通知
        → 夜间 → 静默策略
```
> 关键：状态推进，不是重复提醒

### 闭环③：日报 / 周报沉淀
- **日报**：完成 / 卡点 / 明日重点
- **周报**：趋势 + 薄弱点趋势（时长/完成度/错因）
- **目标**：让家长基于数据决策

### 学习类
- `/作业` `/错题` `/日报` `/周报`

- `/开始` `/结束` `/确认`

### 策略类（家长）
- `/切换计划` `/设置原则`

### 设计原则
> **孩子极简，家长可控。**

## 关键洞察

1. **执行链路 > 内容智能**：系统价值不在于"讲得多好"，而在于"能执行到什么程度"
2. **环境驱动 > 消息提醒**：让音响、台灯、Timer 代替人脑记住节奏
3. **状态机推进**：未确认不是"再提醒一遍"，而是状态升级（轻提醒→家长通知→静默）
4. **角色分离**：同一任务，孩子版和家长版信息密度不同
5. **配置驱动 > 代码硬编码**：用模板+模块+参数实现灵活性
